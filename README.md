# ApiHub Technical README

## 1. 文档目标

本文档整合 ApiHub 项目的三类核心交付物：

1. 数据库 DDL
2. 核心 RESTful API 接口清单
3. DocForge / ApiHub 服务边界与交互协议

默认技术基线：Mysql8、Spring Boot 3.2、Java 21、React 18、Redis 7、RabbitMQ、MinIO、OpenSearch/Elasticsearch。

---

## 2. 数据库 DDL

### 2.1 设计说明

- 采用 PostgreSQL 方言
- 业务主键统一使用 `BIGINT GENERATED BY DEFAULT AS IDENTITY`
- 时间统一使用 `TIMESTAMPTZ`
- 快照、规则、报告等非强结构化内容使用 `JSONB`
- 核心建模方式为“当前实体 + 历史快照”

### 2.2 推荐扩展

```sql
CREATE EXTENSION IF NOT EXISTS pg_trgm;
```

## 4. 服务边界文档

### 4.1 总体原则

DocForge 和 ApiHub 是两个独立协作系统：

- DocForge 面向“源码与产出”
- ApiHub 面向“存储、协作、发布与消费”

目标是让 ApiHub 不绑定源码解析实现，让 DocForge 不承担协作平台复杂性。

### 4.2 DocForge 职责边界

**负责：**

- 多语言源码扫描与 AST 分析
- 注释提取与类型推导
- 统一 IR 建模
- OpenAPI / Markdown / HTML / Postman 生成
- Git 变更检测
- 增量推送与全量推送
- 构建插件与 CLI 集成

**不负责：**

- 用户登录与权限管理
- 文档在线编辑与发布 UI
- Mock 规则维护与运行时 Mock 服务
- 调试代理与测试执行
- 环境变量与密钥托管
- 分享、审计、AI 协作能力

### 4.3 ApiHub 职责边界

**负责：**

- 空间、项目、模块、接口主数据管理
- 用户、成员、角色与权限控制

### 4.5 服务间交互协议

#### 4.5.1 通信方式

- 协议：HTTPS REST
- 内容类型：`application/json`
- 编码：UTF-8
- 幂等控制：`Idempotency-Key`
- 安全：HMAC-SHA256 请求签名

#### 4.5.2 请求头约定

| Header            | 必填  | 说明             |
| ----------------- | --- | -------------- |
| `X-App-Id`        | 是   | DocForge 应用标识  |
| `X-Timestamp`     | 是   | Unix 时间戳       |
| `X-Nonce`         | 是   | 随机串，防重放        |
| `X-Signature`     | 是   | HMAC-SHA256 签名 |
| `Idempotency-Key` | 是   | 幂等键            |

签名原文建议：

```text
HTTP_METHOD
REQUEST_PATH
X-Timestamp
X-Nonce
SHA256(requestBody)
```

#### 4.5.3 推送模式

- 全量推送：首次接入或全量重建
- 增量推送：按 `routeKey` 提交新增、修改、删除
- 删除标记：`documents[].operation=delete`

### 4.6 推送数据格式

#### 4.6.1 顶层 Envelope

```json
{
  "requestId": "push_20260408_001",
  "spaceKey": "platform-rd",
```

### 4.7 ApiHub 接收处理流程

1. 校验 `X-App-Id`
2. 校验时间戳窗口
3. 校验签名
4. 校验 `Idempotency-Key`
5. 写入 `doc_push_record`
6. 定位 `spaceKey/projectKey/moduleKey`
7. 对 `documents` 执行 `upsert/delete`
8. 生成 `api_version`
9. 更新 `api_endpoint.current_revision_id`
10. 写审计日志并异步更新搜索索引

### 4.8 失败与重试约定

| 场景       | ApiHub 行为                  | DocForge 行为 |
| -------- | -------------------------- | ----------- |
| 签名错误     | 返回 `401/403`               | 停止重试，人工处理   |
| 幂等键重复    | 返回 `200` 并标记重复             | 认为成功        |
| 部分文档解析失败 | 返回 `200` 或 `207` + warning | 记录 warning  |
| 服务临时不可用  | 返回 `503`                   | 指数退避重试      |
| 数据约束冲突   | 返回 `409`                   | 停止并输出差异     |

### 4.9 版本协作关系

- DocForge 负责判断源码变化及其文档影响
- ApiHub 负责平台内版本保存、发布、回滚与协作
- DocForge 传入 `commitId/branch/summary`
- ApiHub 落库为 `api_version`、`module_version_tag`、`doc_push_record`

结论：DocForge 是文档生产者，ApiHub 是文档主数据平台与消费平台。
