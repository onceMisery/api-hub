# 🏗️ 全方位落地方案：新一代 API 文档生成 + 管理平台

## 一、现有项目深度分析

### 1.1 Torna —— 企业级 API 文档管理平台

Torna 是一个企业 API 文档解决方案，目标是让文档管理变得更加方便、快捷。它采用协作方式管理和维护项目 API
文档，将不同形式的文档纳入统一维护。Torna 弥补了传统文档生成工具（如 Swagger）的不足，在保持原有功能的前提下丰富并增强了一些实用功能。

**Torna 现有核心功能：**

| 功能领域       | 说明                                                                                                                                                       |
|------------|----------------------------------------------------------------------------------------------------------------------------------------------------------|
| 团队协作       | 空间 → 项目 → 模块三级组织结构                                                                                                                                       |
| 文档管理       | 手动创建/编辑接口，支持 path param/header/body/response                                                                                                             |
| 接口调试       | 内建调试功能，类似简化版 Postman                                                                                                                                     |
| 数据字典       | 统一管理项目中的字典枚举                                                                                                                                             |
| OpenAPI 集成 | 支持 Swagger/OpenAPI 导入                                                                                                                                    |
| 权限控制       | 基础的角色权限体系                                                                                                                                                |
| 推送集成       | smart-doc + Torna 组成行业领先的文档生成和管理解决方案，使用 smart-doc 完成 Java 源代码分析和注释提取生成 API 文档，自动推送到 Torna 平台。推送内容包括 API 名称/作者/路径参数/header/body/response/dict/error code。 |
| 部署方式       | 支持 JAR 包直接运行和 Docker 部署，后端使用 Spring Boot + MySQL。                                                                                                        |

**Torna 的不足：**

- UI 设计较陈旧，无"浏览模式/编辑模式"切换
- 无 Mock 服务能力
- 无版本对比（Diff）功能
- 无自动化测试编排
- 权限粒度较粗
- 无 AI 辅助能力

### 1.2 Smart-doc —— 零侵入 Java API 文档生成工具

Smart-doc 是一个支持 JAVA REST API、Java WebSocket 和 Apache Dubbo RPC 接口文档生成的工具。它基于接口源码分析生成文档，零注解侵入。只需编写
Javadoc 注释，即可生成 Markdown 或 HTML5 文档。Smart-doc 不需要像 Swagger 那样在代码中注入注解。

**Smart-doc 现有核心功能：**

| 功能领域         | 说明                                                                                                    |
|--------------|-------------------------------------------------------------------------------------------------------|
| 零侵入          | 通过分析接口源代码生成全面准确的文档，无需特殊注解或代码修改。完全基于注释生成文档，实现零代码侵入。                                                    |
| 多框架支持        | 支持 Spring MVC、Spring Boot、Spring Boot Web Flux、Feign、JAX-RS。                                          |
| 多输出格式        | 支持多种文档输出格式，包括 Markdown、HTML5、Postman Collection、OpenAPI 3.0 等。                                        |
| RPC 支持       | 支持 Apache Dubbo RPC。                                                                                  |
| gRPC 支持      | Smart-doc 能处理复杂的 gRPC 特性，如双向流、枚举和嵌套消息。                                                                |
| WebSocket 支持 | 从 3.0.7 版本起增加了对 WebSocket 接口的支持。                                                                      |
| 测试脚本         | 可以扫描代码生成 JMeter 性能测试脚本。                                                                               |
| 自定义 Tag      | 由于 Java 原生 javadoc 标签数量有限，smart-doc 自定义了一些 javadoc 标签。                                                |
| 构建集成         | 提供 Maven 和 Gradle 插件，可轻松集成到现有 Java 项目中，支持多种输出格式满足不同团队需求。                                              |
| Torna 推送     | smart-doc + Torna 组成行业领先的文档生成和管理解决方案，使用 smart-doc 完成 Java 源码分析并提取注释生成 API 文档，自动推送到 Torna 企业级接口文档管理平台。 |

**Smart-doc 的不足：**

- 仅支持 Java 生态（不支持 Go/Python/Node.js 等）
- 生成的文档是静态的，缺乏交互性
- 无增量推送和变更检测机制
- 缺乏多语言源码分析引擎

### 3.3 关键创新功能设计

#### 3.3.1 增量变更检测与推送

```java
/**
 * API 变更检测引擎 —— DocForge 独创
 *
 * 对比上次生成的快照与本次分析结果，只推送变更部分
 */
public class ApiDiffEngine {

    /**
     * 快照存储在项目 .docforge/ 目录下
     */
    private static final String SNAPSHOT_DIR = ".docforge/snapshots";

    public ApiChangeSet detectChanges(List<ApiEndpoint> current) {
        List<ApiEndpoint> previous = loadLastSnapshot();
        ApiChangeSet changeSet = new ApiChangeSet();

        for (ApiEndpoint endpoint : current) {
            ApiEndpoint old = findMatch(previous, endpoint);
            if (old == null) {
                changeSet.addAdded(endpoint);
            } else if (!endpoint.deepEquals(old)) {
                changeSet.addModified(endpoint, old);  // 保留新旧对比
            }
        }

        // 检测已删除的接口
        for (ApiEndpoint old : previous) {
            if (findMatch(current, old) == null) {
                changeSet.addRemoved(old);
            }
        }

        saveSnapshot(current);  // 保存当前快照
        return changeSet;
    }
}

/**
 * 增量推送器 —— 只推送变更部分到 ApiHub
 */
public class IncrementalPusher {

    public PushResult push(ApiChangeSet changeSet, PushConfig config) {
        PushPayload payload = PushPayload.builder()
                .projectToken(config.getToken())
                .added(changeSet.getAdded())
                .modified(changeSet.getModified())
                .removed(changeSet.getRemoved())
                .changelog(changeSet.generateChangelog())
                .timestamp(Instant.now())
                .gitCommitId(getGitCommitId())    // 关联 Git 提交
                .gitBranch(getGitBranch())
                .build();

        return apiHubClient.pushIncremental(payload);
    }
}
```

#### 3.3.2 多语言分析器插件体系

```java
/**
 * 语言分析器 SPI 接口 —— 支持通过插件扩展新语言
 */
public interface LanguageAnalyzer {

    /** 支持的语言标识 */
    String language();  // "java", "go", "typescript", "python"

    /** 分析源码目录，返回统一 IR */
    AnalysisResult analyze(Path sourceRoot, DocForgeConfig config);

    /** 支持的框架列表 */
    List<String> supportedFrameworks();
}

/**
 * Java 分析器实现（核心）
 */
@AutoService(LanguageAnalyzer.class)
public class JavaLanguageAnalyzer implements LanguageAnalyzer {

    // 注册框架解析器
    private final Map<String, FrameworkResolver> resolvers = Map.of(
            "spring-mvc", new SpringMvcResolver(),
            "spring-webflux", new WebFluxResolver(),
            "dubbo", new DubboResolver(),
            "grpc", new GrpcResolver(),
            "jax-rs", new JaxRsResolver(),
            "websocket", new WebSocketResolver(),
            "solon", new SolonResolver()   // NEW! 支持国产框架
    );

    @Override
    public AnalysisResult analyze(Path sourceRoot, DocForgeConfig config) {
        // 1. 加载依赖源码 (自动分析依赖树)
        DependencyTree depTree = DependencyTreeLoader.load(sourceRoot);

        // 2. AST 解析
        List<CompilationUnit> units = parseSourceFiles(sourceRoot, depTree);

        // 3. 框架识别与接口提取
        String framework = detectFramework(units);
        FrameworkResolver resolver = resolvers.get(framework);

        // 4. 类型推导 (泛型、嵌套、递归)
        TypeInferenceEngine typeEngine = new TypeInferenceEngine(depTree);

        // 5. 生成统一 IR
        return resolver.resolve(units, typeEngine, config);
    }
}
```

**后端权限实现:**

```java
/**
 * 统一权限注解
 */
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface RequirePermission {
    ResourceType resource();           // SPACE, PROJECT, MODULE, ENDPOINT

    PermissionAction action();         // VIEW, EDIT, DELETE, MANAGE

    String resourceIdParam() default "id";  // 从参数中取资源 ID
}

/**
 * 权限拦截器 —— AOP 实现
 */
@Aspect
@Component
public class PermissionAspect {

    @Autowired
    private PermissionService permissionService;

    @Around("@annotation(requirePermission)")
    public Object checkPermission(ProceedingJoinPoint pjp, RequirePermission rp) throws Throwable {
        Long userId = SecurityContext.getCurrentUserId();
        Long resourceId = extractResourceId(pjp, rp.resourceIdParam());

        boolean hasPermission = permissionService.check(
                userId, rp.resource(), resourceId, rp.action()
        );

        if (!hasPermission) {
            throw new ForbiddenException("无权执行此操作");
        }
        return pjp.proceed();
    }
}

/**
 * 权限服务 —— 支持继承链查询
 */
@Service
public class PermissionService {

    public boolean check(Long userId, ResourceType type, Long resourceId, PermissionAction action) {
        // 1. 先查缓存
        String cacheKey = String.format("perm:%d:%s:%d:%s", userId, type, resourceId, action);
        Boolean cached = redisTemplate.opsForValue().get(cacheKey);
        if (cached != null) return cached;

        // 2. 向上遍历权限链
        //    Module → Project → Space
        MemberRole role = findEffectiveRole(userId, type, resourceId);
        boolean result = role.hasPermission(action);

        // 3. 缓存结果 (5分钟)
        redisTemplate.opsForValue().set(cacheKey, result, Duration.ofMinutes(5));
        return result;
    }

    private MemberRole findEffectiveRole(Long userId, ResourceType type, Long resourceId) {
        // 先查当前层级的显式角色
        MemberRole directRole = memberRepository.findRole(userId, type, resourceId);
        if (directRole != null) return directRole;

        // 递归向上查找父级权限
        Long parentId = getParentResourceId(type, resourceId);
        ResourceType parentType = type.getParentType();
        if (parentType != null && parentId != null) {
            return findEffectiveRole(userId, parentType, parentId);
        }

        return MemberRole.NONE;
    }
}
```

```java
    @Autowired
    private DebugProxyService proxyService;

    @PostMapping("/send")
    @RequirePermission(resource = ResourceType.PROJECT, action = PermissionAction.VIEW)
    public DebugResponse sendRequest(@RequestBody DebugRequest request) {
        // 1. 解析环境变量
        Environment env = environmentService.getById(request.getEnvId());
        String fullUrl = resolveVariables(env.getBaseUrl() + request.getUrl(), env.getVariables());

        // 2. 合并全局 Header 和请求 Header
        Map<String, String> mergedHeaders = new LinkedHashMap<>();
        mergedHeaders.putAll(env.getGlobalHeaders());
        mergedHeaders.putAll(request.getHeaders());

        // 3. 执行变量替换 (支持 {{variable}} 语法)
        String body = resolveVariables(request.getBody(), env.getVariables());

        long startTime = System.nanoTime();

        HttpResponse<String> response = httpClient.send(
                HttpRequest.newBuilder()
                        .uri(URI.create(fullUrl))
                        .method(request.getMethod(), BodyPublishers.ofString(body))
                        .headers(flattenHeaders(mergedHeaders))
                        .timeout(Duration.ofSeconds(30))
                        .build(),
                BodyHandlers.ofString()
        );

        long durationMs = (System.nanoTime() - startTime) / 1_000_000;

        // 5. 构造响应
        DebugResponse debugResponse = DebugResponse.builder()
                .statusCode(response.statusCode())
                .headers(response.headers().map())
                .body(response.body())
                .durationMs(durationMs)
                .bodySize(response.body().getBytes().length)
                .build();

        // 6. 异步保存调试历史
        debugHistoryService.saveAsync(request, debugResponse);

        return debugResponse;
    }
```

```java
public class MockGatewayController {

    @RequestMapping(method = {RequestMethod.GET, RequestMethod.POST,
            RequestMethod.PUT, RequestMethod.DELETE})
    public ResponseEntity<String> handleMock(
            @PathVariable String projectToken,
            HttpServletRequest request) {

        // 1. 定位项目和接口
        Module module = moduleService.findByPushToken(projectToken);
        String path = extractMockPath(request);
        String method = request.getMethod();

        ApiEndpoint endpoint = endpointService.match(module.getId(), method, path);
        if (endpoint == null) {
            return ResponseEntity.notFound().build();
        }

        // 2. 查找匹配的 Mock 规则
        List<MockRule> rules = mockRuleService.findActive(endpoint.getId());
        MockRule matchedRule = matchRule(rules, request);

        String responseBody;
        int responseCode;
        Map<String, String> responseHeaders;

        if (matchedRule != null) {
            // 3a. 使用匹配到的规则
            responseBody = mockTemplateEngine.render(matchedRule.getResponseBody());
            responseCode = matchedRule.getResponseCode();
            responseHeaders = matchedRule.getResponseHeaders();

            // 模拟延迟
            if (matchedRule.getDelayMs() > 0) {
                Thread.sleep(matchedRule.getDelayMs());
            }
        } else {
            // 3b. 基于接口定义自动生成 Mock 数据

        responseHeaders.forEach(headers::set);
        headers.set("X-Mock-By", "ApiHub");

    }
```

```java
@Service
public class ApiDiffService {

    /**
     * 对比两个版本的接口差异
     */
    public DiffResult compare(Long endpointId, String versionA, String versionB) {
        ApiVersion snapshotA = versionRepo.findByEndpointAndVersion(endpointId, versionA);
        ApiVersion snapshotB = versionRepo.findByEndpointAndVersion(endpointId, versionB);

        EndpointSnapshot a = parseSnapshot(snapshotA.getSnapshotData());
        EndpointSnapshot b = parseSnapshot(snapshotB.getSnapshotData());

        DiffResult result = new DiffResult();

        // 1. 基本信息对比
        if (!Objects.equals(a.getUrl(), b.getUrl())) {
            result.addChange(DiffItem.breaking("url", a.getUrl(), b.getUrl()));
        }
        if (!Objects.equals(a.getHttpMethod(), b.getHttpMethod())) {
            result.addChange(DiffItem.breaking("httpMethod", a.getHttpMethod(), b.getHttpMethod()));
        }

        // 2. 参数对比 (递归对比嵌套结构)
        diffParameters(result, "request", a.getParameters(), b.getParameters());

        // 3. 响应对比
        diffParameters(result, "response", a.getResponses(), b.getResponses());

        // 4. Header 对比
        diffHeaders(result, a.getHeaders(), b.getHeaders());

        // 5. 分类: Breaking / Non-Breaking / Added / Removed
        result.classify();

        return result;
    }

    /**
     * 模块级别对比 (批量)
     */
    public ModuleDiffResult compareModuleVersions(Long moduleId, String tagA, String tagB) {
        ModuleVersionTag a = moduleVersionTagRepo.find(moduleId, tagA);
        ModuleVersionTag b = moduleVersionTagRepo.find(moduleId, tagB);

        ModuleDiffResult result = new ModuleDiffResult();

        Map<String, EndpointSnapshot> mapA = parseModuleSnapshot(a);
        Map<String, EndpointSnapshot> mapB = parseModuleSnapshot(b);

        // 找出新增、删除、变更的接口
        for (String key : Sets.union(mapA.keySet(), mapB.keySet())) {
            if (!mapA.containsKey(key)) {
                result.addNew(mapB.get(key));
            } else if (!mapB.containsKey(key)) {
                result.addRemoved(mapA.get(key));
            } else {
                DiffResult diff = compare(mapA.get(key), mapB.get(key));
                if (diff.hasChanges()) {
                    result.addModified(key, diff);
                }
            }
        }

        return result;
    }
}
```

```java

@Service
public class TestSuiteExecutor {

    @Autowired
    private DebugProxyService proxyService;

    @Autowired
    private ScriptEngine scriptEngine;  // Nashorn / GraalJS

    /**
     * 执行测试集合
     */
    public TestExecutionResult execute(Long suiteId) {
        TestSuite suite = testSuiteRepo.findById(suiteId);
        List<TestStep> steps = testStepRepo.findBySuiteId(suiteId, Sort.by("stepOrder"));
        Environment env = environmentService.getById(suite.getEnvId());

        // 全局变量上下文 (用于步骤间传递数据)
        VariableContext context = new VariableContext(env.getVariables());

        TestExecutionResult execution = new TestExecutionResult();
        execution.setSuiteId(suiteId);
        execution.setTotalSteps(steps.size());
        execution.setStartTime(Instant.now());

        for (TestStep step : steps) {
            if (!step.isEnabled()) continue;

            StepResult stepResult = executeStep(step, context, env);
            execution.addStepResult(stepResult);

            // 如果步骤失败且配置了"失败停止"，则中止
            if (stepResult.isFailed() && suite.isStopOnFailure()) {
                break;
            }
        }

        execution.setEndTime(Instant.now());
        execution.calculateSummary();

        // 保存执行记录
        testExecutionRepo.save(execution);

        return execution;
    }

    /**
     * 执行单个测试步骤
     */
    private StepResult executeStep(TestStep step, VariableContext context, Environment env) {
        StepResult result = new StepResult();
        result.setStepName(step.getName());

        try {
            // 1. 变量替换 (URL, Headers, Body 中的 {{variable}})
            String url = context.resolve(step.getUrl());
            Map<String, String> headers = context.resolveMap(step.getHeaders());
            String body = context.resolve(step.getBody());

            // 2. 执行前置脚本 (可修改请求参数)
            if (step.getPreScript() != null) {
                ScriptContext scriptCtx = new ScriptContext(context, url, headers, body);
                scriptEngine.execute(step.getPreScript(), scriptCtx);
                url = scriptCtx.getUrl();
                headers = scriptCtx.getHeaders();
                body = scriptCtx.getBody();
            }

            // 3. 发送请求
            DebugRequest request = DebugRequest.builder()
                    .method(step.getMethod())
                    .url(url)
                    .headers(headers)
                    .body(body)
                    .envId(env.getId())
                    .build();

            long startTime = System.nanoTime();
            DebugResponse response = proxyService.send(request);
            result.setDurationMs((System.nanoTime() - startTime) / 1_000_000);
            result.setResponse(response);

            // 4. 执行后置脚本 (可提取变量)
            if (step.getPostScript() != null) {
                ScriptContext scriptCtx = new ScriptContext(context, response);
                scriptEngine.execute(step.getPostScript(), scriptCtx);
            }

            // 5. 提取变量 (JSONPath)
            if (step.getExtractors() != null) {
                for (VariableExtractor extractor : step.getExtractors()) {
                    String value = extractValue(response, extractor);
                    context.setVariable(extractor.getVarName(), value);
                }
            }

            // 6. 执行断言
            if (step.getAssertions() != null) {
                for (Assertion assertion : step.getAssertions()) {
                    AssertionResult ar = executeAssertion(assertion, response, context);
                    result.addAssertionResult(ar);
                }
            }

            result.setStatus(result.allAssertionsPassed() ? "passed" : "failed");

        } catch (Exception e) {
            result.setStatus("error");
            result.setErrorMessage(e.getMessage());
        }

        return result;
    }

    /**
     * 基于 JSONPath 提取变量
     */
    private String extractValue(DebugResponse response, VariableExtractor extractor) {
        return switch (extractor.getSource()) {
            case "body" -> JsonPath.read(response.getBody(), extractor.getPath()).toString();
            case "header" -> response.getHeaders().get(extractor.getPath());
            case "status" -> String.valueOf(response.getStatusCode());
            default -> throw new IllegalArgumentException("Unknown source: " + extractor.getSource());
        };
    }
}
```

## 七、核心竞争力总结

| 维度      | Torna (现有) | Smart-doc (现有) | **ApiHub + DocForge (新方案)**         |
|---------|------------|----------------|-------------------------------------|
| 文档生成    | 不生成，接收推送   | Java 单语言       | **多语言 (Java/Go/TS/Python)，插件化**     |
| 推送机制    | 全量覆盖       | 全量推送           | **增量推送 + 变更检测 + Git 关联**            |
| 团队协作    | 基础角色       | 无              | **三级权限继承 + 细粒度操作控制**                |
| UI 体验   | 传统表单       | 静态 HTML        | **双模式 (编辑/阅读) + Dark Mode + 现代 UI** |
| 接口调试    | 基础调试       | 调试页面           | **全功能调试 + 跨域代理 + 环境变量**             |
| Mock 服务 | ❌ 无        | ❌ 无            | **智能 Mock (规则 + AI 生成)**            |
| 版本管理    | ❌ 无        | ❌ 无            | **Git-like 版本快照 + Diff 对比**         |
| 自动化测试   | ❌ 无        | JMeter 脚本      | **Test Suite 编排 + 断言 + CI/CD**      |
| AI 能力   | ❌ 无        | ❌ 无            | **6 大 AI 辅助功能 (可选)**                |
| 全文搜索    | 基础搜索       | 无              | **Elasticsearch 全文搜索**              |
| 部署方式    | JAR/Docker | 无需部署           | **Docker Compose 一键部署 + K8s**       |

---

这套方案从 **文档定义 → 源码生成 → 增量推送 → 协作管理 → Mock 模拟 → 接口调试 → 自动化测试 → 版本追踪 → AI 辅助** 实现了
API 的**全生命周期管理**，在 Torna + Smart-doc 的基础上实现了质的飞跃。建议按照路线图分 Phase 迭代开发，MVP
阶段聚焦核心流程打通（生成→推送→管理→调试），后续逐步叠加高级功能。
